# Sudoku Studio 骨架规划

## 1. 产品目标
- 做一个纯前端数独游戏，零依赖，可直接部署到 GitHub Pages。
- 风格参考 iOS：简洁、圆角、轻玻璃感、浅色层次。
- 兼顾新手与进阶玩家：既有教程，也有高亮与笔记工具。
- 技术选择以“浏览器原生现代能力”为主，不为了框架而引入框架。

## 2. 页面结构
- 顶部状态栏：难度、计时、错误数、当前模式（普通/教程）。
- 主游戏区：左侧 9x9 棋盘，右侧固定操作观察面板。
- 操作观察面板：当前格信息、候选芯片、全局观察数字、工具按钮、教程解析。
- 侧边教程区：可点击开始的教程关卡卡片。

## 3. 游戏核心元素
- 棋盘生成：随机生成完整解，再按难度挖空，保证唯一解。
- 难度等级：简单/中等/困难（给定数字数量不同）。
- 选中高亮：行、列、宫联动高亮。
- 数字占线显示：仅在点击数字键盘时触发；点击棋盘数字不触发占线提示。
- 冲突提示：重复数字冲突即时标红。
- 颜色区分：题目给定数字和玩家新填数字使用不同颜色。
- 笔记模式：为单元格记录候选数。
- 撤销/重做：支持操作回滚。
- 提示系统：优先裸单/隐藏单，兜底给出正确数字。
- 提示交互：先解释为什么这一步成立，再允许玩家决定是否直接应用。
- 键盘支持：方向键移动、1-9 输入、N 切笔记、H 提示。
- 进度持久化：自动保存到 `localStorage`，刷新后恢复。

## 4. 教程关卡（内置）
- 教程 1：裸单（Naked Single）
- 教程 2：隐藏单（Hidden Single）
- 教程 3：锁定候选（Locked Candidates）
- 教程 4：X-Wing

## 5. 技术拆分
- `index.html`: 页面结构与交互入口。
- `styles.css`: 视觉风格、响应式布局、状态样式。
- `scripts/data.js`: 常量、难度配置、教程关卡数据。
- `scripts/core.js`: 生成器、求解器、候选计算、冲突检查、存档校验。
- `scripts/ui.js`: 棋盘渲染、右侧观察面板、教程详情和消息渲染。
- `scripts/app.js`: 应用状态、交互逻辑、事件绑定、存档恢复。
- `README.md`: 使用与 GitHub Pages 部署说明。

## 6. 为什么这样构建
- 这是一个静态前端游戏，不需要服务端渲染和复杂数据流框架。
- 原生 `ES Modules` 已足够现代，也天然兼容 GitHub Pages。
- 先把逻辑层、渲染层、应用层拆开，比先上框架更能解决当前真正的问题：棋盘逻辑复杂、教程系统会继续增长、UI 需要持续微调。
