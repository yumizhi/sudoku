# Sudoku Studio

一个可直接部署到 GitHub Pages 的纯前端数独游戏，采用浏览器原生 `ES Modules + HTML + CSS`。

## 方案选择

- 不强行上 Vue/React。
- 直接使用现代浏览器原生模块，避免构建链、依赖安装和额外部署复杂度。
- 对这个项目更合适，因为它是前端本地游戏，核心难点在棋盘逻辑、状态管理和 UI 布局，不在服务端或复杂工程化。
- 结果是：保留现代化结构、保持可扩展、可直接部署到 GitHub Pages，并且继续维持“打开即跑”。

## 三步完成情况

### 第一步：骨架规划
- 已在 `docs/BLUEPRINT.md` 完成产品骨架与模块规划。
- 页面结构：顶部状态栏 + 左侧棋盘 + 右侧操作观察面板 + 教程关卡区。
- 功能元素：难度、计时、占线高亮、候选观察、笔记、提示、撤销重做、冲突检查、键盘快捷键、自动存档。
- 技术骨架：`data / core / ui / app` 四层拆分。
- 视觉方向：简洁浅色玻璃界面，强调可读性和桌面端可操作区常驻。

### 第二步：按骨架开发
- `index.html`：完整页面结构。
- `styles.css`：iOS 风样式、响应式布局、棋盘状态色。
- `scripts/data.js`：常量与教程关卡数据。
- `scripts/core.js`：数独生成、求解、候选与校验等纯逻辑。
- `scripts/ui.js`：DOM 查询、棋盘渲染、状态面板渲染。
- `scripts/app.js`：应用状态、事件绑定、存档、提示和流程控制。
- 已实现：随机生成完整解 + 挖空生成题面（唯一解校验）。
- 已实现：数字输入、占线显示（仅点击数字盘触发）。
- 已实现：冲突标红、错误计数、检查、提示（裸单/隐藏单/兜底）。
- 已实现：解释优先的提示面板，可先看逻辑说明，再决定是否应用这一步。
- 已实现：笔记模式、撤销重做、键盘输入支持。
- 已实现：教程关卡模式（含 X-Wing 关卡）。
- 已实现：题目给定数字与玩家新填数字分色显示。

### 第三步：细化与检查
- 移动端适配：手机上数字盘与工具栏自动改为多列。

## 本地运行

推荐在项目根目录启动一个本地静态服务器。

如果系统有 `python3`，直接运行：

```bash
python3 -m http.server 4173
```

然后打开：

```text
http://localhost:4173
```

说明：
- 现在项目使用浏览器原生 `ES Modules`。
- GitHub Pages 部署没有问题。
- 但部分浏览器直接打开本地 `index.html` 时，模块导入会被本地文件策略限制，所以本地开发更适合走静态服务器。

## 部署到 GitHub Pages

1. 将当前仓库推送到 GitHub。
2. 打开仓库 `Settings` -> `Pages`。
3. `Build and deployment` 里选择：
   - `Source`: `Deploy from a branch`
   - `Branch`: `main`（或你的默认分支） / `/(root)`
4. 保存后等待 1~3 分钟，GitHub 会给出访问链接。

## 快捷键
- `1-9`: 输入数字
- `Backspace/Delete/0`: 擦除
- `N`: 切换笔记模式
- `H`: 请求提示
- `方向键`: 移动选中格
